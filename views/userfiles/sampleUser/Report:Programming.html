I implemented clean urls for all of my files (including the user specific ones)by having express run get and post to urls of my choosing.                                               I can load any userdata by deserializing, which saves some space as well.

I dynamically generate my dashboard and note pages using ejs. My dashboard queries the database and loads all of the user’s files                                                     into a table (its actually using li in the html) where it sets each to trigger that file to load when clicked. I also use ejs to send back                                                   error messages on my forms (login, signup,create note).

I do all of my client/server interactions through form data, most of which is happening hidden from the user. The main part of my                                               server implementation in both time and content has been having the user’s documents properly save and load. After a number of                                             different trial and errors, I have this working by storing the canvas data to a url which (when save is pressed) sends that url back to                                                   the server to be processed into a png that is saved in view/userdata to be loaded when the note is next called (immediately, since save                                         redirects to the note and reloads the data). I get the content editable data the same way, using.innerHTML and sending the text when                                             save is pressed to an html file. This lets the user store their information off of the public file system, and lets me load it relatively easily                                         using some requests to the user’s file database(userFiles.db) and passing the stored information to the correct locations.
